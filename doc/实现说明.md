# 简化版斯诺克实现说明

## 实现目标

当前版本基于 **Cocos Creator 3.8.6 + TypeScript**，实现了一个可继续扩展的斯诺克原型。当前主线已经从早期的关卡草案收敛到以下方向：

- 首屏先进入模式选择首页
- 提供休闲模式与专家模式
- 鼠标与触摸统一输入
- 逻辑、物理、规则、UI 分离
- 先保证稳定可玩，再逐步补充规则和表现

当前主界面的视觉参考以 `doc/main.png` 为准。

## 当前已落地内容

- 主界面
  - 模式选择首页
  - 休闲模式入口
  - 专家模式入口
  - 音效开关占位
- 游戏界面
  - 顶部 HUD：得分、BREAK、模式名、状态提示
  - 底部 HUD：力度条、辅助线开关、操作提示
  - 暂停、重开、返回主页等弹层按钮
- 核心玩法
  - 白球拖拽瞄准
  - 松手出杆
  - 球与球碰撞
  - 球与库边碰撞
  - 落袋判定
  - 统一停球检测
  - 白球落袋犯规与自动重置
  - 进球计分与 BREAK 结算
- 对局配置
  - 休闲模式：偏短局体验
  - 专家模式：标准斯诺克开局
  - 具体球组与目标配置统一放在 `SnookerConfig.ts`
- 结算
  - 本地最高分记录
  - 单局结束结算
  - 规则提示弹层

## 模块职责

### `SnookerConfig.ts`

- 统一维护设计分辨率、桌台尺寸、袋口位置、物理参数
- 统一维护休闲模式与专家模式的配置
- 维护标准斯诺克开局布局数据

### `SnookerTypes.ts`

- 统一定义球类型、模式、阶段、关卡结构、球状态结构
- 为后续扩展更复杂的对局流程预留类型

### `PointerInput.ts`

- 封装鼠标与触摸输入
- 上层只接收统一的开始 / 移动 / 结束事件

### `SnookerPhysics.ts`

- 负责简化二维物理
- 包括：
  - 线性摩擦减速
  - 库边反弹
  - 球体碰撞
  - 落袋检测
  - 全部停球判断

### `SnookerRules.ts`

- 负责本杆结算
- 包括：
  - 进球得分
  - 白球落袋犯规
  - BREAK 变化
  - 状态文案

### `SnookerGame.ts`

- 作为游戏总入口
- 负责：
  - 构建模式选择首页、游戏界面和结算界面
  - 组织回合流程
  - 调用输入、物理、规则模块
  - 创建程序化桌台、球体和运行时 UI

### `UiFactory.ts`

- 负责场景节点复用
- 统一处理圆角面板、圆形节点和文本节点的创建

### `SnookerTheme.ts`

- 维护当前版本使用的视觉主题色盘与材质倾向
- 供主界面、HUD、桌台和按钮统一复用

### `TextureSkinner.ts`

- 负责把资源目录中的正式 PNG 贴图接到运行时节点上
- 让程序化布局也能复用最终资源

## 运行方式

### 方式一：直接打开当前目录

如果 Cocos Creator 3.8.6 可以识别该目录为项目，直接打开项目并进入 `assets/scenes/Main.scene`。

### 方式二：导入到一个空的 Cocos Creator 3.8.6 2D 项目

如果本地 Creator 对空仓库识别不稳定，建议这样做：

1. 新建一个 **Cocos Creator 3.8.6 2D TypeScript 项目**
2. 将当前仓库中的 `assets/` 整个复制进去
3. 打开 `assets/scenes/Main.scene`
4. 如果脚本组件没有自动挂上，就把 `SnookerGame` 挂到 Canvas 下的 `GameRoot` 或任意空节点上

## 当前取舍

- 为了优先可玩，当前仍使用自定义轻量物理，而不是 Creator 内置 2D 物理
- 音效按钮只保留状态开关，没有接入音频资源
- 没有实现职业斯诺克复杂规则，如先红后彩、彩球复位、旋转球等
- 关卡式流程仍保留在数据和脚本层，后续可以继续扩展，但不作为当前主入口

## 推荐下一步

1. 把 `SnookerGame.ts` 中的首页搭建、比赛流程和结算流程继续拆成更独立的模块
2. 给模式选择和结算逻辑补更完整的状态流转
3. 接入音效、震动、进球粒子
4. 如需更真实手感，再逐步替换成 Creator 2D Physics 或 Box2D 方案
