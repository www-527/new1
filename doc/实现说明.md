# 简化版斯诺克实现说明

## 实现目标

当前版本基于 **Cocos Creator 3.8.6 + TypeScript**，实现了一个可以直接继续扩展的简化版斯诺克原型，重点对齐了 `doc/design.txt` 和 `doc/design.jpg` 中的这些方向：

- 休闲、清爽、偏移动端的横屏体验
- 单人玩法，规则简化，但保留瞄准、蓄力、击球、进球得分的核心乐趣
- 鼠标与触摸统一输入
- 逻辑与表现分离，优先保证“稳定可玩”

## 已完成内容

- 主界面
  - 练习模式
  - 闯关模式
  - 上一关 / 下一关
  - 音效开关占位
- 游戏界面
  - 顶部 HUD：得分、BREAK、模式/关卡、剩余杆数
  - 底部 HUD：力度条、状态提示
  - 暂停 / 重开 / 返回主页
- 核心玩法
  - 白球拖拽瞄准
  - 松手出杆
  - 球与球碰撞
  - 球与库边碰撞
  - 落袋判定
  - 统一停球检测
  - 白球落袋犯规与自动重置
  - 进球计分与 BREAK 结算
- 关卡
  - 1 个练习布局
  - 10 个示例关卡配置
- 结算
  - 练习完成结算
  - 闯关成功 / 失败结算
  - 本地最高分记录

## 模块职责

### `SnookerConfig.ts`

- 统一维护设计分辨率、桌台尺寸、袋口位置、物理参数
- 提供练习模式与 10 个关卡的布局数据

### `SnookerTypes.ts`

- 统一定义球类型、模式、阶段、关卡结构、球状态结构

### `PointerInput.ts`

- 封装鼠标与触摸输入
- 上层只接收统一的开始 / 移动 / 结束事件

### `SnookerPhysics.ts`

- 负责简化二维物理
- 包括：
  - 线性摩擦减速
  - 库边反弹
  - 球体碰撞
  - 落袋检测
  - 全部停球判断

### `SnookerRules.ts`

- 负责本杆结算
- 包括：
  - 进球得分
  - 白球落袋犯规
  - BREAK 变化
  - 状态文案

### `SnookerGame.ts`

- 作为游戏总入口
- 负责：
  - 构建主界面、游戏界面、结算界面
  - 组织回合流程
  - 调用输入、物理、规则模块
  - 创建程序化桌台和球体表现

## 运行方式

### 方式一：直接打开当前目录

如果 Cocos Creator 3.8.6 可以识别该目录为项目，直接打开项目并进入 `assets/scenes/Main.scene`。

### 方式二：导入到一个空的 Cocos Creator 3.8.6 2D 项目

如果你本地 Creator 对空仓库识别不稳定，建议这样做：

1. 新建一个 **Cocos Creator 3.8.6 2D TypeScript 项目**
2. 将当前仓库中的 `assets/` 整个复制进去
3. 打开 [Main.scene](/E:/new1/assets/scenes/Main.scene)
4. 如果脚本组件没有自动挂上，就把 `SnookerGame` 挂到 Canvas 下的 `GameRoot` 或任意空节点上

## 当前取舍

- 为了优先可玩，当前没有接入 Creator 内置 2D 物理，而是使用自定义轻量碰撞
- 音效按钮目前只保留状态开关，没有接入音频资源
- 没有实现职业斯诺克复杂规则，如先红后彩、彩球复位、旋转球等
- 结算和关卡以第一版验证玩法为主，便于后续继续拆分扩展

## 推荐下一步

1. 把 `SnookerGame.ts` 中的“界面搭建”和“比赛流程”继续拆成单独模块
2. 给关卡系统补真正的关卡选择页
3. 接入音效、震动、进球粒子
